  Pacotes(bpl) e conexões com Banco de Dados
Publicado por ActiveDelphi : Sexta, Março 23, 2007 - 08:51 AM GMT-3 (2124 leituras internas) Continuação... (9101 bytes) 5 Comentários
“Ah! Mais um texto sobre pacotes do Delphi ou modularização de aplicações.”. Espero que não seja só isso que as pessoas pensem ao lerem este artigo. Todo mundo sabe criar aplicações exe, você vai ao Delphi, cria uma nova aplicação, compila e pronto. Mas... e pacotes?

Quando comecei a ler sobre eles e tentar construir algo não achava grandes documentos que oferecesse uma abordagem simples, mas explicativa do assunto.

Vamos começar pelo princípio, para podermos chegar ao que importa: a conexão com o banco de dados usando pacotes.

Para criar uma aplicação modularizada se faz necessário, é claro, de um exe para poder carregar os pacotes. Então, crie uma nova aplicação igualmente as normais que você já é acostumado: File/New/Application.


Figura 1 - Criando uma nova aplicação

Salve a sua aplicação normalmente e nas opções do projeto (Menu Project, Options), você deve marcar o item "Build with runtime packages" e limpar o campo logo abaixo, deixando igual à imagem que segue:


Figura 2 - Opções do Projeto

Compile sua aplicação e verá que o tamanho do executável ficou bem pequeno. Lembre-se dessa tela, pois você terá que voltar nela mais adiante.

Agora feche o projeto do executável, vá ao menu File/New/Other e procure por Package.


Figura 3 - Criando um pacote

Irá aparecer uma tela igual a essa:


Figura 4 - Janela do pacote criado

Os pacotes possuem os Contains, que são as units, forms, dm e etc que ele possui e os Requires, que são outros pacotes que ele utiliza.

Vamos adicionar uma unit ao pacote e nela criar uma classe para conectar ao banco de dados. Eu uso e recomendo a library do Zeos para trabalhar com banco de dados, mas existem outras formas. Implemente da forma que precisar ou necessitar.

Veja abaixo o fonte que fiz para criar uma classe que irá se conectar ao banco de dados. Você poderá fazer à sua maneira:

unit uConnection;

interface
uses ZConnection,SysUtils;

type
  TConnection = class
  private
    FConnection: TZConnection;
  public
    constructor Create;
    destructor Destroy;override;
    property Connection: TZConnection read FConnection;
  end;
var
  Connection : TConnection;
implementation
{ TConnection }
constructor TConnection.Create;
begin
  FConnection := TZConnection.Create(nil);
  with FConnection do
    begin
      HostName := 'localhost';
      Port := 3306;
      User := 'artigo';
      Password := 'artigo';
      Protocol := 'mysql-5';
      Database := 'ARTIGO';
    end;
  FConnection.Connected := TRUE;
end;

destructor TConnection.Destroy;
begin
  FConnection.Disconnect;
  FConnection.Free;
end;

initialization
  Connection := TConnection.Create;
 
finalization
  Connection.Free;
end.

Essa classe fará a conexão com a base e a manterá ativa até que o aplicativo seja finalizado.

Salve e compile o mesmo. No local em que o arquivo foi compilado, você encontrará um arquivo com a extensão bpl. Este é o pacote de conexão, e também terá um dcp, que é para usar agora no projeto1, que tem o exe.

Feche todas os projetos abertos e abra o primeiro projeto, que possui o executável, para podermos modificar algumas coisas nele.

Vá novamente às opções do projeto e modifique-as como abaixo:


Figura 5 - Alterando as opções do primeiro projeto

Feche a tela acima e na tela principal do sistema ou até mesmo no projeto dpr, adicione ao uses o dbconnection, para que ele seja inicializado junto com a aplicação.

Salve e compile-o novamente.

Agora vamos criar um form para conectar a base e seu respectivo pacote.

Feche os projetos, vá ao menu File/New/Other e Packages. Criado o pacote, crie dentro dele um novo form, também no menu File/New/Form. No pacote do formulário, você deve adicionar o pacote de conexão, pois você depende dele.


Figura 6 - Pacote de Formulário

No formulário coloquei o seguinte:


Figura 7 - Interface do Formulário

Vou mostrar somente fazendo a conexão com a base e executando uma consulta. O botão abrir ficou:

ZQuery1.Close;
ZQuery1.SQL.Text := 'SELECT * FROM CLIENTES';
ZQuery1.Connection := Connection.Connection;
ZQuery1.Open;

Nas uses, adicione a unit de conexão para poder usar o Connection. Ao final da unit, registrei a classe do formulário para poder usar ele no exe:

initialization
  RegisterClass(TForm1);

finalization
  UnRegisterClass(TForm1);

Usei no exemplo o banco de dados Mysql, mas como disse, isso serve para qualquer banco. Abaixo esta o script de criação da tabela que uso nessa aplicação.

CREATE TABLE `clientes` (
  `CODIGO` int(11) NOT NULL auto_increment,
  `NOME` varchar(40) default NULL,
  `DATA_NASCIMENTO` date default NULL,
  PRIMARY KEY  (`CODIGO`),
  UNIQUE KEY `CODIGO` (`CODIGO`)
) ENGINE=InnoDB DEFAULT CHARSET=latin1;

Fecho todos os projetos abertos no delphi e abro o projeto do executável, para inserir o código que chamará o pacote, que é feito da seguinte forma:

procedure TForm1.Button1Click(Sender: TObject);
var
  PackageModule: HModule;
  AClass: TPersistentClass;
begin
  PackageModule := LoadPackage('formulario.bpl');
  if PackageModule <> 0 then
  begin
    AClass := GetClass('TForm1');
    if AClass <> nil then
      with TComponentClass(AClass).Create(Application)
        as TCustomForm do
      begin
        ShowModal;
       Free;
      end;
    UnloadPackage(PackageModule);
  end;
end;
